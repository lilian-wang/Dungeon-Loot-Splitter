Logic Explanation

The loot array is the single source of truth because it stores every item's name, value, and quantity, and all total calculations come directly from this data. Quantity was added so total loot could be computed from value x quantity instead of treating each item as a single unit.

All total calculations occur inside updateUI(), which recomputes the total loot value, and determines the amount each party member receives. The renderLoot() function is responsible only for rebuilding the visible loot list from the current state.

Whenever the user adds loot, removes loot, or changes the party size, the state changes and updateUI() recalculates totals, rebuilds the loot list, updates the split value, and controls which sections are visible. This ensures the interface always reflects the current application state.

State also determines what the user can do. If the loot array is empty or the party size is invalid, the Split Loot button is disabled and results stay hidden.

Conditional logic in addLoot() prevents invalid items from entering the array, and conditional logic in updateUI() prevents invalid party sizes from producing results. This keeps the interface consistent with the underlying data at all times.
