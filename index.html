Logic Explanation

I use global Loot array to store all loot items. Each time the user submits the Add Loot form, a new item containing the loot name and value is pushed into this array so the data persists across the entire application.

I use loops in two places: a traditional for loop in renderLoot(), to display every loot item on the page, and another for loop in getTotalLoot() to add up all loot values and calculate the running total.

I use conditional logic inside addLoot() to validate the loot name and loot value before adding an item to the array. I also use conditions in renderLoot() to check whether any loot exists; if the array is empty, the interface displays a message instead of a list. In splitLoot(), I use conditions to ensure the party size is valid and that loot has been entered before performing the split. These checks prevent invalid input and handle edge cases directly inside the interface.
